role para começar

O problema

Cada sistema, uma ilha.

WhayDesk, e-mail, chat interno, controle financeiro, licitação, planilhas — e outros. Ferramentas boas, mas isoladas. Cada uma resolve um pedaço e reinventa tudo de novo: autenticação, permissão, auditoria, interface e banco de dados.

  • ✕
    Logins que não conversamO usuário entra várias vezes; a TI mantém vários cadastros da mesma pessoa.
  • ✕
    Permissões duplicadasCada sistema tem seu próprio jeito de dizer quem pode o quê — sem padrão.
  • ✕
    Dados em silosO mesmo cliente existe em 4 bancos, com 4 versões da verdade.
  • ✕
    Integração no braçoExporta, importa, copia e cola. Planilha vira cola entre sistemas.

A virada

Um Core. Muitos módulos.

O LUMEN inverte a lógica. Em vez de N sistemas isolados, uma plataforma: o Core cuida do que é comum a todos, e cada domínio vira um módulo independente que se encaixa nele.

  • ✓
    Identidade única e permissão centralUm login para tudo; RBAC granular no formato modulo.recurso.acao.
  • ✓
    Multiempresa nativaDados particionados por empresa (company_id) — um ambiente, vários clientes.
  • ✓
    Módulos plugáveis, banco isoladoCada módulo tem seu Docker e seu banco — modularidade real, mas integrada pelo Core.
  • ✓
    Uma linguagem visual sóTodos os módulos compartilham o mesmo design — parecem o mesmo produto.

Como se compõe

Reaproveitar, não recomeçar.

Nada de bom é jogado fora. A Matriz de Composição de Domínio define, módulo a módulo, o que vem de cada sistema maduro e como isso é fundido no padrão LUMEN — capacidade e regra, não cópia de código.
